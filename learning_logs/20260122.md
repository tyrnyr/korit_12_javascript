  - ES6 상의 단축어 지원해주는 앱니다. / 자동완성 지원 snippets은 재사용 가능한 코드 조각을 의미합니다. 그래서 이전에 쓴 코드랑 일치한거 막 미리 보여주고 하는 편리한 애라고 생각해주시면 됩니다.

# JavaScript의 역사
1990년대에 브랜던 아이크가 처음에 모카(Mocha)라는 이름으로 지었었습니다. 그 다음에 livescript로 갔다가 javascript가 되었습니다. 기본적으로는 브라우저에서 실행되는 프로그래밍 언어로 발전했습니다.

옛날에는 각 브라우저 상에서 구동되는 언어가 달랐기 때문에 A브라우저에서 실행되는 사이트가 B브라우저에서는 안되는 일이 자주 발생해서 Script 언어에 대한 표준이 필요해졌습니다.

# ECMA Script
European Computer Manufacturers Association이라는 유럽 컴퓨터 제조업체 연합에서 JavaScript의 표준화에 대한 작업이 시작되었습니다. ECMA-262란 이름으로 시작되었었습니다. 즉 JS 자체가 아니라 다른 언어도 된다는 뜻입니다. 하지만 사실상 JS를 표준화하기 위해서 만들어진 스크립트 표준입니다. ECMA Script의 축약어가 ES고, ES6

# JS 문법
## 변수와 상수
- 변수(Variable) - 데이터를 담는 바구니 : 데이터/객체/함수를 사용하겠다고 선언하고 이름을 붙이는 것.
- 상수(Const) - 변화하지 않는 값을 가지는 변수
## 변수 선언 형식
```js
var     x     = "Hello World!";
선언자 변수명  = 데이터
```
### 변수 명명 규칙
1. 문자, 숫자, _, $를 포함하여 만들 수 있다.
2. 변수명은 숫자로 시작할 수 없다.
3. JS 예약어는 사용할 수 없다.(예를 들면 console과 같은 변수명으로 쓸 수 없다는 뜻입니다)
4. 변수명은 대소문자를 구분한다.

- ES에서 변수명과 함수명은 camel case 사용 / 클래스명과 생성자명에는 Pascal case를 권장합니다.

### 선언자의 종류
1. var    - 변수용
2. let    - 변수용
3. const  - 상수용

- var 선언자
  - JS는 Java와는 달리 데이터 타입에 상관없이 변수를 사용할 때 var / let을 사용할 수 있습니다. 현재에는 오류를 최소화하기 위해 주로 let을 사용하는데, 이미 개발된 많은 JS 프로그램이 var로 되어있기 때문에 혹시 유지보수 팀으로 들어가게 될 경우 var 선언자를 보게될 확률이 높습니다.

```js
var x = 5;
var y = 6;
var z = x + y;

```
02_variable.js

```js
var x = 5;
var y = 6;
var z = x + y;
console.log(z);

var x = "안녕하세요";      // 여기서 오류가 발생해야 정상인거 같습니다.
z = x + y;
console.log(z);
```
그런데 오류가 발생하지 않는 점을 봤을 때 var 선언자를 사용하게 되면 변수명이 겹치더라도 오류가 나지 않기 때문에 원하는 결과값이 다르게 발생할 수 있습니다.

동일한 변수명으로 새로운 변수가 선언되면, 기존 코드에서 동일한 이름으로 사용되던 변수에 할당된 값이 사라지므로, 해당 변수를 참조하고 있는 프로그램에서 오류가 발생할 가능성이 있습니다.

그래서 ES6 이후로 let 선언자가 생겼고, 오늘날에는 let을 쓸것을 권장하고 있습니다.

- let 선언자
얘는 동일한 변수명을 허용하지도 않고, 나중에 선언된 변수를 미리 등록(메모리에 저장)해두지도 않습니다.

- const 선언자
final이랑 같긴 합니다. 선언 시점에 값을 할당한 후에 변수로 재선언 및 재대입이 불가능합니다.

- Java의 경우에 final 키워드는 되게 조심스럽게 사용했었습니다.
- 하지만 JS의 경우에는 좀 더 적극적으로 const를 사용합니다.

- JS의 개발철학상 변수는 브라우저에서 값을 바꿀 수 있기 때문에 모두가 동일한 결과값을 가지기가 힘듭니다. 하지만 상수로 선언하게 되면 값이 바뀌지 않기 때문에 다수의 사용자가 동일한 결과값을 얻을 수 있다는 점 때문에 const의 사용을 장려합니다.

그래서 함수는 const로 선언하는 편입니다.

* 주의 : 함수의 결과값을 const로 집어넣는다는 의미가 아닙니다.

```js
let name1 = "김일";

```
표준적으로는 String 입력할 때 ''를 쓰는 편입니다.
  - HTML에서 태그 내에 값 입력할 때 ""가 나왔기 때문에 헷갈리지 않도록 ''를 쓰는 편입니다.
- Number
숫자 타입에 대한 구분이 없습니다. 그냥 따옴표 안쓰고 쓰면 됩니다.
- Boolean
- Undefined
  - 얘는 좀 특이한데 undefined 자체가 데이터 값이자 데이터 자료형입니다. 단어 그대로 정의되지 않았다는 의미입니다.

```js
let car;      // 선언은 했는데 아직 초기화가 되지 않았다면
console.log(car);     // undefined로 콘솔에 찍힙니다.
```
- null
  - 얘도 데이터 값이자 자료형입니다. 없음을 의미하는 것은 동일한데, 의미상의 차이가 존재하기 때문에 구분할 필요가 있습니다.
    - undefined의 경우에는 웹 브라우저에서 JS를 실행할 때 초기화가 되지 않은 변수에 대해 자동으로 undefiend를 리턴해주기 때문에 개발자가 아니라 브라우저 엔진이 할당하는 값입니다.
    - 반면에 null의 경우는 개발자가 명시해서 사용합니다. 아직 어떤 값을 할당할지 모르거나 어떤 자료형의 값이 들어올지 모를 때 개발자가 명시적으로 null을 대입해둘 수 있습니다.
```js
let car = null;
```
  - null은 JS의 메모리를 효율적으로 관리하고 싶을 때 사용할 수 있습니다.
  - 그러니까 일단 변수를 선언해서 값 집어넣고 있다가, 더이상 해당 변수가 사용되지 않을 때 맨 마지막으로 null값을 넣어버리게 되면 메모리에서 해당 변수를 더이상 참조하지 않게 됩니다.
  - 그러면 더이상 안쓰이는 변수에 대한 메모리 누수가 존재하지 않게 된다는 점에서 특정 시점에 null값을 대입해주는 방식을 통해 사용성을 향상시킬 수 있습니다.

- Symbol - ES6에 추가된 앤데 나중에 수업하겠습니다.

## Objects
### Object
- 추후 좀 자세히 다루기는 할 겁니다.
- 기본적으로 JS의 모든 것은 객체에 해당합니다. 그런데 선언 방식이 좀 달랐죠.
```js
const person = {
  firstName: 'John',
  lastName: 'Doe',
  age: 50,
  eyeColor: 'blue'
}
```
key - value properties로 이루어져있다, 라고 했습니다.

```js
person["age"] = 51;   // 객체명["key값"] = 재대입할 데이터;
person.age = 51;    // 객체명.key값 = 재대입할 데이터;

console.log(person['firstName']);
console.log(person.firstName);
```

### Array
- 하나 이상의 데이터를 하나의 변수에 담아뒀습니다. 여러 값을 하나의 단일 참조(single reference)로 저장할 수 있습니다(index를 통해). 배열에 저장된 데이터의 특징은 순서를 갖는다는 점입니다.

```js
const cars = ['현대', '기아', '혼다'];

console.log(cars[0]);
console.log(cars[1]);
console.log(cars[2]);

cars[2] = '벤츠';
```

그러면 초기화한 값에서 추가하거나 혹은 삭제하는 방법에 대한 문제가 생길 수 있습니다. 이는 배열 내장함수를 통해서 학습하겠습니다.

## typeof
- JS에서는 typeof 연산자를 활용하여 변수의 자료형을 알아낼 수 있습니다.
- Java에서는 얘 필요 없었습니다. 애초에 선언할 때 저희가 알고 선언해야 하니까요.

```js
let name = '김일';
console.log(typeof name);   // string
```
instanceof의 경우에는 a instanceof b의 형태로 써야만 했습니다. 그런데 하나만 쓴다는 점에서 연산자라고 생각하기 어려울 수 있습니다. 하지만 함수라고 이야기하지 않는 이유는 typeof(name);이 아니기 때문입니다.
